# Build Status Template

Use this format for every automation build.

## Idea

`<manual task or automation idea>`

## Business Value

`<time saved, revenue impact, risk reduction, quality improvement>`

## Systems Involved

`<Open WebUI, n8n, HubSpot, Gong, Evergrowth, AirOps, APIs, Codex>`

## Data Required

`<records, transcripts, signals, docs, logs, sensitivity>`

## Workflow Status

`<idea | loop spec | design | build | test | debug | QA | approval | active | paused>`

## Test Status

`<not started | payloads ready | dry-run passed | failed | fixed | ready>`

## Risks

`<security, privacy, data quality, operational, commercial>`

## Owner

`<person responsible>`

## Next Action

`<single next step>`

## Ready To Activate

`yes/no`

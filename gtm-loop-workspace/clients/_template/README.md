# Client Template

Copy this folder to `clients/<client-slug>/` for a real client.

## File Map

| File | Purpose |
| --- | --- |
| `context.md` | Client objective, stakeholders, scope, decisions, and constraints. |
| `systems.md` | System inventory and ownership. |
| `hubspot.md` | HubSpot objects, fields, workflows, and write rules. |
| `gong.md` | Gong data, signals, transcript use, and sensitivity notes. |
| `airops.md` | AirOps workflows, prompts, inputs, outputs, and approval gates. |
| `n8n.md` | n8n workflows, credentials by name, triggers, retries, and active state. |
| `workflow-inventory.md` | Automation inventory across systems. |
| `field-mapping.md` | Source-to-target field map with transforms and evidence. |
| `endpoint-matrix.md` | APIs, webhooks, methods, payloads, auth model, and status. |
| `configuration-inventory.md` | Setting names, owners, and usage only. No values. |
| `open-questions.md` | Questions that block scope, build, validation, or risk decisions. |
| `build-handoff.md` | Codex/n8n/AirOps implementation seed. |
| `validation-notes.md` | Test runs, fake payloads, expected results, and production readiness. |

## Minimum Useful Client Dossier

Fill these first:

1. `context.md`
2. `systems.md`
3. `open-questions.md`
4. `workflow-inventory.md`
5. `build-handoff.md`

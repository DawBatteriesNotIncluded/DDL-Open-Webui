# How To Use Open WebUI Knowledge RAG

## Recommended Collections

- Loop Specs: upload `loops/*.md`.
- Build System: upload `build-system/*.md`.
- Integration Notes: upload `knowledge/*-notes.md`.
- Workflow Templates: upload `workflow-templates/*.md`.
- Prompts: upload `prompts/*.md`.
- Evals: upload `evals/*.md`.
- Operating Docs: upload selected `docs/*.md`.

## What Belongs In Knowledge

- Playbooks.
- Templates.
- Checklists.
- Fake schemas and examples.
- API notes without secrets.
- Process docs.

## What Does Not Belong In Knowledge

- Real API keys.
- `.env` files with values.
- Sensitive customer data.
- Patient-identifiable data.
- Private sponsor data.
- Raw Gong transcripts unless explicitly approved.

## Retrieval Practice

Ask agents to cite which knowledge documents influenced the answer when debugging or evaluating loops.

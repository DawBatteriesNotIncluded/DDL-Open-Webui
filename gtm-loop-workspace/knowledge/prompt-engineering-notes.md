# Prompt Engineering Notes

## Practical Prompt Pattern

1. Define role and task.
2. Provide context and source boundaries.
3. Define required output schema.
4. Define decision criteria.
5. Require uncertainty and assumptions.
6. Require approval gates.
7. Forbid unsupported claims.

## GTM Prompt Rules

- Ground outputs in provided context.
- Label inference.
- Prefer concise seller-ready output.
- Include next action.
- Include risks and missing data.

## Run ID Rule

Use `RUN-0001`, `RUN-0002`, and so on. Do not use dates as IDs.

## What To Log

- Task model changes or migrations.
- Investigations that change client understanding.
- Workflow designs or architecture packs.
- Build handoffs.
- Validation runs.
- Debugging sessions with root-cause findings.
- Approval requests and outcomes.

## What Not To Log

- Every small note edit.
- Raw logs.
- Secrets or customer records.
- Long transcripts.

# Investigate Workflow

Use for client systems, integrations, APIs, repos, workflows, and mismatched claims.

## Steps

1. Read `llm-wiki/current-client.md` and the client folder.
2. State the exact question.
3. List evidence sources before inspecting deeply.
4. Capture findings as `Confirmed`, `Confirmed mismatch`, `Unknown`, or `Inferred`.
5. Update the narrowest file: system notes, workflow inventory, endpoint matrix, field mapping, config inventory, or open questions.
6. Create or update `build-handoff.md` only when implementation is clear.

## Output Shape

| Bucket | Finding | Evidence | Next Action |
| --- | --- | --- | --- |
| Confirmed | `TBD` | `TBD` | `TBD` |
| Confirmed mismatch | `TBD` | `TBD` | `TBD` |
| Unknown | `TBD` | Evidence missing | `TBD` |

## Stop When

- The build decision is clear, or
- The next question needs owner/system access, or
- More searching is unlikely to change the answer.

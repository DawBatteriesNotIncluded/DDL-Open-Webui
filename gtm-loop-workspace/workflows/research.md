# Research Workflow

Use when a tool, repo, API, vendor capability, pricing, security, or implementation pattern needs source-grounded research.

## Steps

1. Create `research/<topic-slug>/` from `research/_template/`.
2. Add the question to `notes.md`.
3. Inspect primary sources first when current facts matter.
4. Log inspected sources in `sources.md`.
5. Put working analysis in `notes.md`.
6. Put concise conclusions in `brief.md`.
7. Add evidence cards when auditability matters.
8. Update `research/index.md`.

## Closeout

- Mark facts, inference, unknowns, and recommendation separately.
- Do not copy full pages or long source text.
- Add only client-relevant conclusions to the client folder or `llm-wiki/`.
